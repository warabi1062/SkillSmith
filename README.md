# SkillSmith

Claude Code のスキルを、意図した設計パターン通りに作り続けるためのツール。

---

## 背景

Claude Code でスキルを作り込んでいくと、ある問題に直面する。

Orchestrator スキルを中心に Sub Agent を組み合わせる設計パターンを一度確立しても、新しいスキルを AIに生成させるたびに「本当にこれは自分の設計思想と同じ構造になっているか？」という確信が持てない。自由なテキストで書かれた `.md` ファイルは、AI にとって解釈の余地が大きすぎる。

結果として、スキルが増えるほど設計の一貫性が失われていく。

---

## SkillSmith が解決すること

SkillSmith は、スキルの設計パターンを **スキーマとして形式化** し、GUI 上でスキルを組み立てさせることで、設計の準拠を構造的に保証する。

- AIへの指示ではなく、**ツールの構造そのもの**がパターンへの準拠を強制する
- Orchestrator とそれが呼ぶ Sub Agent の関係を視覚的に定義できる
- 設計した構造から、Claude Code が読み込める `.md` ファイルを自動生成（export）する

「正しく書いてもらう」から「正しくしか書けない」へ。

---

## 設計思想

詳細については[./docs/reference.md](./docs/reference.md)を参照

- **Orchestrator 中心設計**: Orchestrator スキルが全体のフローを管理し、各 Step で Sub Agent を起動する
- **単一責任の Sub Agent**: 各 Sub Agent は単一の作業のみを行う。Sub Agent 同士は直接通信しない
- **ファイルパスを契約点に**: Sub Agent の成果物はファイルに保存し、ファイルパスのみを Orchestrator に返す。次の Sub Agent へはそのパスを渡す
- **コンテキストの肥大化防止**: 成果物をファイルに委ねることで、Main Agent のコンテキストをクリーンに保つ

このパターンに沿ったスキルのみを SkillSmith は生成できる。
